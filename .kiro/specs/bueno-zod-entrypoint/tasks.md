# Implementation Plan: bueno-zod-entrypoint

## Overview

Add a `@coveo/bueno/zod` export path that re-exports `zod/mini` APIs. This involves adding `zod` as a dependency, creating a source file, extending the esbuild configuration with two new bundle functions, and updating `package.json` exports. The implementation is additive — existing build outputs remain untouched.

## Tasks

- [x] 1. Add Zod dependency and create source file
  - [x] 1.1 Add `zod` dependency to `packages/bueno/package.json`
    - Add `"zod": "^4.0.0"` to the `dependencies` field
    - Run `pnpm install` to resolve the dependency
    - _Requirements: 3.1_

  - [x] 1.2 Create the Zod entrypoint source file `packages/bueno/src/zod.ts`
    - Single line: `export * from "zod/mini";`
    - This is the facade that re-exports all public Zod Mini APIs
    - _Requirements: 1.2, 3.2_

- [x] 2. Extend esbuild configuration

    - Add both new functions to the `Promise.all` call alongside existing builds
    - Do NOT modify existing build functions (`nodeCjs`, `nodeEsm`, `browserEsm`, `browserUmd`)

## Notes

- Tasks marked with `*` are optional and can be skipped for faster MVP
- Each task references specific requirements for traceability
- The design explicitly states property-based testing does not apply to this feature (build tooling + re-export facade)
- The browser bundle (`cdn/zod.esm.js`) is CDN-only and NOT included in the npm package
- Existing build functions must remain completely untouched — only additive changes to `esbuild.mjs`
- TypeScript declarations are auto-generated by the existing `tsc` build step (no tsconfig changes needed)
