# Use layouts

[An Atomic layout](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-layout-section--default) is an out-of-the-box way of positioning Atomic components in a webpage.
Instead of writing CSS, you can define a parent layout and a number of child sections
(for example, search, facets, pagination, etc.). Child sections can also serve as parents as they can nest other sections.
Each section can nest only specific components, as shown in the diagram below.

It's important to carefully define the layout structure.
However, some optional sections can be left out.

> **Tip**
>
> Atomic currently includes only the search layout.
> Atomic will include more layouts in the future for agent panels, commerce pages, etc.


## [The Search Layout](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-search-layout--default)

![A diagram that shows the search layout](../assets/images/layouts.jpg)


Here are the possible sections for your search layout:

* `search`: an optional section to be used at the top of the layout.
It will be aligned with the results.
  * [`atomic-search-box`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-search-box--default)
* `facets`: an optional section that will display all facet types in the left column. If you omit this section, the `search` and `main` sections will take up the whole space.
  * [`atomic-facet-manager`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-facet-manager--default)
  * all facet types
* `main`: a parent section which takes up most of the space.
* `status`: a section at the top of the `main` one, containing all components that display or allow users to interact with the search status.
  * [`atomic-breadbox`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-breadbox--default)
  * [`atomic-query-summary`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-query-summary--default)
  * [`atomic-refine-toggle`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-refine-toggle--default)
  * [`atomic-sort-dropdown`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-sort-dropdown--default)
  * [`atomic-did-you-mean`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-did-you-mean--default)
* `results`: a section that contains the result list and alternative states.
  * [`atomic-result-list`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-result-list--default)
  * [`atomic-query-error`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-query-error--default)
  * [`atomic-no-results`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-no-results--default)
* `pagination`: the bottom section containing pagination components.
  * [`atomic-load-more-results`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-load-more-results--default)
  * [`atomic-pager`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-pager--default)
  * [`atomic-results-per-page`](https://static.cloud.coveo.com/atomic/v3/storybook/index.html?path=/story/atomic-results-per-page--default)
