The `atomic-ipx-body` component provides a structured layout for IPX (In-Product Experience) interfaces. It includes designated slots for header, body, and footer content, with built-in styling for a consistent user experience.

## Usage

### Basic Structure

The component defines three named slots for organizing content:

```html
<atomic-ipx-body>
  <div slot="header">
    <!-- Header content like title, close button -->
  </div>
  <div slot="body">
    <!-- Main content, automatically scrollable -->
  </div>
  <div slot="footer">
    <!-- Footer content like action buttons -->
  </div>
</atomic-ipx-body>
```

### Embedded Mode

When `isOpen` is undefined (default), the component operates in embedded mode without visibility classes:

```html
<atomic-ipx-body>
  <!-- Content slots -->
</atomic-ipx-body>
```

### Modal Mode

Use the `isOpen` property to control visibility with animation classes:

```html
<atomic-ipx-body is-open="true">
  <!-- Content slots -->
</atomic-ipx-body>
```

### Hiding the Footer

Set `displayFooterSlot` to `false` to hide the footer area:

```html
<atomic-ipx-body display-footer-slot="false">
  <div slot="header">Header</div>
  <div slot="body">Body content without footer</div>
</atomic-ipx-body>
```

## Features

- **Structured Layout**: Predefined slots for header, body, and footer content
- **Scrollable Body**: The body section automatically becomes scrollable when content overflows
- **Customizable Scrollbar**: Styled scrollbar that matches the design system
- **Semantic HTML**: Uses proper semantic elements (article, header, footer) for accessibility
- **Animation Support**: Emits `animationEnded` event for animation coordination

## Implementation Notes

This component is typically used internally by other IPX components like `atomic-ipx-modal` and `atomic-ipx-embedded` to provide a consistent content structure.
