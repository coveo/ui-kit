The `atomic-result-text` component renders the value of a string result field.

This component is used within `atomic-result-template` components inside the result list:

```html
<atomic-search-interface>
  ...
  <atomic-result-list>
    <atomic-result-template>
      <template>
        <atomic-result-text field="title"></atomic-result-text>
        <atomic-result-text field="excerpt"></atomic-result-text>
        <atomic-result-text field="author" default="no-author"></atomic-result-text>
      </template>
    </atomic-result-template>
  </atomic-result-list>
  ...
</atomic-search-interface>
```

### Basic Usage

To display a specific result field:

```html
<atomic-result-text field="excerpt"></atomic-result-text>
```

### With Default Value

When a field might not have a value, you can provide a fallback:

```html
<atomic-result-text field="brand" default="no-brand-available"></atomic-result-text>
```

### With Highlighting

By default, the component will highlight search terms in supported fields (`title`, `excerpt`, `firstSentences`, `printableUri`  and `summary`  ). You can disable this behavior:

```html
<atomic-result-text field="excerpt" should-highlight="false"></atomic-result-text>
```

### Supported Fields

The component looks for fields in this order:
1. First in the Result object properties
2. Then in the `result.raw` object
