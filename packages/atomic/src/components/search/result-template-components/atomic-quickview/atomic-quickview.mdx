# atomic-quickview

The `atomic-quickview` component renders a button which the end user can click to open a modal box containing a preview about a result.

The `atomic-quickview` is not meant to replace the `atomic-result-link` to access an item in a result template; it has certain limitations (for example, custom styles and embedded images/links may not work as expected in an `atomic-quickview`).

## Usage

<Canvas of={QuickviewStories.Default} />

### In an atomic-search-interface

```html
<atomic-search-interface>
  <atomic-result-list>
    <atomic-result-template>
      <template>
        <atomic-quickview></atomic-quickview>
      </template>
    </atomic-result-template>
  </atomic-result-list>
</atomic-search-interface>
```

## Properties

### `sandbox`

The `sandbox` attribute to apply to the quickview iframe.

The quickview is loaded inside an iframe with a [`sandbox`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/iframe#attr-sandbox) attribute for security reasons.

This attribute exists primarily to protect against potential XSS attacks that could originate from the document being displayed.

By default, the sandbox attributes are: `allow-popups allow-top-navigation allow-same-origin`.

**Important:** `allow-same-origin` is not optional, and must always be included in the list of allowed capabilities for the component to function properly.

**Type:** `string`  
**Default:** `'allow-popups allow-top-navigation allow-same-origin'`

#### Example

```html
<atomic-quickview
  sandbox="allow-scripts allow-popups allow-top-navigation allow-same-origin"
></atomic-quickview>
```
